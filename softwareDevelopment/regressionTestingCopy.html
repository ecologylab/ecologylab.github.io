  1. Buld JavaDocs. Look at the error messages that are generated. Resolve problems. Also add documentation as needed.
  2. Launch the news collection from the browser. After it's running fine for 5 minutes:
    • Save the information space without history.
    • Load the saved version of the information space without history.
    • Save the information space with history.
    • Load the saved version of the information space with history.
  3. From the browser, launch a re:mix with 2-4 searches and websites that are of interest to you. Once it runs o.k. for some time:
    • Save the information space without history.
    • Load the saved version of the information space without history.
    • Save the information space with history.
    • Load the saved version of the information space with history.
  4. Within eclipse, run a search with 2-4 keyword queries that are of interest to you. Once it runs o.k. for some time:
    • Save the information space without history.
    • Load the saved version of the information space without history.
    • Save the information space with history.
    • Load the saved version of the information space with history.
  5. Build the ecologylab.jar and cF.jar files. Sign both jars and (if necessary) re-sign all-pdf.jar. Then, from the web interface in both Internet Explorer and Firefox:
    • Open a new (blank) composition space and ensure that it starts up properly.
    • Run the news collection and ensure it starts up properly.
    • Seed a new session with 2-4 queries consisting of only sites and ensure that it starts up properly.
    • Seed a new session with 2-4 queries consisting only of keywords and ensure that it starts up properly.
    • Seed a new session with 2-4 queries consisting of half sites, half keywords. Ensure that it starts up properly.
  6. After launching from the web interface, try navigating to a container or container's hyperlink. Ensure that the page loads properly in the browser. Do this for both Internet Explorer and Firefox.
  7. Use drag and drop to bring elements in from a web page. Do this both with our extension installed in Firefox, and without it in Internet Explorer and Firefox. Save them to a file. Start a new session and open the file to ensure it opens properly.
  8. Create several new text elements in an empty space. Save the space. Restart combinFormation and restore the contents of the space to ensure that it opens properly.
  9. Edit some existing elements in a space. Save the space. Restart combinFormaiton and restore the contents of the space.
  10. Edit the metadata of some existing elements in the space. Save the composition, restart combinFormation, and reload the space. Ensure that the metadata displays and that the changes were preserved.
  11. Express interest in metadata terms using the in-context-slider. Save the composition, restart combinFormation, and reload the space. Ensure that the metadata term's interest levels have been preserved.
  12. Run any large search or collection for 8 hours, or until it crashes or becomes impossibly slow. In the console output, occasional debugging information will be shown indicating number of elements in the candidate pools and the free memory. If possible, notice these values at the time when the session stopped being productive, or when the 8 hours was up (whichever is first). The program should be able to run for at least the 8 hours.
