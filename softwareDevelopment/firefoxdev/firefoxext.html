Firefox Extensions

Firefox provides an extensions mechanism which allows 3rd parties to modify the browser's appearance and abilities. For the most part the extensions tend to modify the browser's UI and the various scripts which control its behavior. This is generally referred to as the applications "chrome". This is a similar concept to skinning, but offers more capabilities than just changing the appearance of the application.

Arbitrary XUL applications can be run inside of the Firefox browser, through the use of a Chrome url. A Chrome URL is special protocal avialable on the Mozilla platform. It functions in much the same way as a normal URL, except that rather than pointing to a HTML document, a Chrome URL points to a XUL application. Such Applications are run as trusted code, and can access all aspects of the browser which are exposed through XPCOM interfaces. Additionally, these applications can also use a special XUL device called an overlay to replace, modify, or enhance existing browser XUL applications.

There is very little difference between an XUL application and a Firefox extension. The only tangible difference is the way in which the XUL is distributed. For a normal XUL application, the application is accessed directly via its chrome URL, something like chrome://www.asd.com/myXULAPP.xul. The browser then executes the XUL application, subsequently unloads it when a different URL is entered into the applications. It perhaps best to think of a nromal XUL application as a special kind of web page, that only works in in the Mozilla browsers.

An extension on the other hand, is packaged a distributed via a XPI file. This is an archive file which contains the XUL code, and several RDF/XML documents which tell the browser how to install the application. Unlike a normal XUL application, extensions are installed locally, loaded when the browser starts, and continue execution until the browser is shut down, or a user turns it off through a UI component.

In addition to being distributed in this way, many Firefox extensions utilize an XUL construct called an overlay, overlays allow you to replace, modify, and add additional XUL code into an existing XUL file or application. This technology enables us to change the browser behavior and appearance without having to submit and maintain code inside the Mozilla or Firefox projects.

Rather than try to create and maintain an extension tutorial I will now redirect you to the primary place to find all things related to Firefox extension development. The MozillaZine Dev Extensions is the best place to get started.

The primary tutorial I suggest is the one at roachfiend.com. This is a very clear, easy to follow, step by step guide to get you up and running.
