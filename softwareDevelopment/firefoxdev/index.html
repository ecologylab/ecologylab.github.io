Overview

It is important to begin by providing some background information concerning The Firefox browser. As you are likely aware, the Firefox browser is created and maintained as part of the Mozilla project. Although it is partially true to say that Firefox is an upgraded version of the Mozilla browser, the reality is that Firefox is a new XUL based application which utilizes the underlying Mozilla code base. In fact if you look at the about box for your Firefox browser it will include the Mozilla version that your firefox install is built on.

The Mozilla code base provides the underlying core functionality of the browser, exposed through XPCOM objects, as well as the Java Script Interpeter, XUL parsing and interpreters, HTML parsing and visualization. Perhaps it is best to describe Mozilla as a platform itself. Mozilla can essential be thought of as an operating system abstraction, which provides events, graphics context, threading, I/O and networking. The Mozilla Platform is designed to run on a number of operating systems, including Microsoft Windows, Linux, and Unix.

Although it is possible to pull down a Mozilla base distribution, and begin creating your own applications directly, it is much easier to utilize the Firefox or Mozilla browser to run your application, thus removing the need to distribute and maintain large binary distributions. Applications can be more easily constructed using the XUL language, and running from within Firefox using a chrome URL.

Basically XUL utilizes XML, CSS, HTML, and Java Script, to access XPCOM components which access abstracted operating system functions. For example it is easy to create a window using the following XUL

<window id="findfile-window" title="Find Files" orient="horizontal" xmlns="http://www.mozilla.org/keymaster/gatekeeper/there.is.only.xul"> ... </window>

When Firefox executes the XUL file containing this code, it finds the appropriate XPCOM objects for creating windows, gets a reference to the XPCOM interface, and request a new window object. The XPCOM objects are responsible for talking to the actual operating system and actually creating the window. Firefox then uses the XPCOM interface for the window to enable Java Script access to the newly created window object.

XUL applications are only limited by the abilities of Java Script and the availability of XPCOM objects. In the case that you need functionality which is not available in the Mozilla base XPCOM objects, it is possible to create your own XPCOM interfaces and objects. This type of work is outside the scope of this document, however for interested parties, IBM's Developer Works has a nice series of articles covering XPCOM, available here.
