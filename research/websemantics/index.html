By web semantics, we refer to descriptive information, including significant attributes, features, representations, and relationships, of entities on the web, for people to use in their tasks. Examples of web semantics include the gist of a news report, price of a product, reviews of a book, and portrait of a friend.

Web semantics provide essential and valuable contextual information, allowing users to make sense of, assess, act on, and make decisions about underlying entities. For example, semantics for a scholarly article, including the author list, publication venue, references, and citations, are all essential for its readers to assess how important the work is and understand how it connects to the field. In another example, a user shopping online looks at semantics of a product, which can include not only its price, brand, specs, and pictures, but also reviews, related products, and feeds from the user's social networks, to make the purchasing decision. Web semantics are especially important when users need to relate, compare, and synthesize across multiple entities, for example, in curation. Without contextual web semantics, users will have to view information from different sources in separate browser windows, and keep track of linked information with limited working memory, which results in problems of getting lost and distracted.

Web semantics can take various structures, formally defined by data models, and countless presentations, such as lists, tables, maps, and templated web pages. This makes it difficult for computer applications to obtain and use valuable web semantics. Prior approaches, such as the Semantic Web and microdata, rely on websites to actively provide and maintain semantics, and have not been widely used. Facebook's Open Graph and Twitter's Cards are supported by many websites, but only address limited categories of web semantics.

The Interface Ecology Lab conducts research in various areas involving web semantics. We developed practical method for deriving image-text clippings as surrogates of web pages, to provide contextual semantics more than the page title. We built BigSemantics, an open source architecture, language, and software library, for representing, extracting, and presenting web semantics. With BigSemantics, developers can author wrappers, using the meta-metadata language, to define and reuse data models for different types of web semantics, and specify presentation semantics to display web semantics in usable forms. BigSemantics comes with a non-trivial repository of wrappers, addressing popular websites and common use cases involving searches, products, digital libraries, creative work, and social media. The BigSemantics Service supports acquisition and storage of web semantics in web scale with a distributed processing architecture. An example web application, the Metadata In-Context Explorer (MICE), shows how BigSemantics supports exploratory browsing, a paradigm that allows users to serendipitously encounter diverse and novel information in the same context.
